# Brainstorm Sections

This reference describes what makes a great brainstorm requirements document.

## Hard floor

- **Summary** — what is being proposed, in 1-3 lines.
- **Requirements** (R-IDs) — what must be true about the proposed thing.

## Include when material

- Problem Frame
- Key Decisions
- Actors
- Key Flows
- Acceptance Examples
- Success Criteria
- Scope Boundaries
- Dependencies / Assumptions
- Outstanding Questions
- Sources / Research
