# Architecture Diagram Update

`/ce:diagram:update` keeps the architecture diagram in sync with what was actually built after each review cycle. It diffs the current codebase against `architecture-constraints.md`, shows what changed, and applies confirmed changes to the diagram and constraints file.

## Workflow

### Phase 1: Gather Inputs

Collect the diff inputs:
- Read `architecture-constraints.md` from the project root. If it does not exist, stop and print: "No architecture-constraints.md found. Run `/ce:diagram` first to establish the baseline."
- Check whether `architecture.excalidraw` exists in the project root. If it does not exist, note this — the update will create it from scratch after applying the diff.
- Collect the git diff if available: run `git diff HEAD --name-only` (suppress errors if not a git repo)

### Phase 2: Invoke Diagram Differ

Invoke agent `excalidraw-diagrams:diagram:diagram-differ` with:
- The path to `architecture-constraints.md`
- The project root path (for file tree scan)
- The git diff output (if available)

### Phase 3: Handle Empty Diff

If the agent returns `NO_CHANGES`, print:

"Diagram is up to date. No changes found between architecture-constraints.md and the current codebase."

Stop — no further action needed.

### Phase 4: Print Diff Summary

Parse the structured diff returned by the agent and print a human-readable summary:

```
I found the following changes since the last diagram update:

ADDED ({N} components):
  + ProfileScreen (Screen) — lib/features/profile/presentation/profile_screen.dart

REMOVED ({N} components):
  - OldAuthService

RENAMED ({N}):
  UserBloc -> ProfileBloc

LAYER VIOLATIONS ({N}):
  AnalyticsHelper: expected Service, found in Screen layer

NEW FLOWS ({N}):
  ProfileBloc -> UserRepository

REMOVED FLOWS ({N}):
  HomeBloc -> AuthService
```

### Phase 5: Confirm Before Applying

Ask the user (use `AskUserQuestion` in Claude Code, `request_user_input` in Codex, `ask_user` in Gemini):

"Apply these {N} changes to architecture.excalidraw and architecture-constraints.md, and log them to architecture-changelog.md?"

If the user declines, stop without writing anything.

### Phase 6: Apply Changes

**Update `architecture-constraints.md`:**
- For each ADDED component: append a new row to the Layer Ownership table
- For each REMOVED component: remove its row from the Layer Ownership table
- For each RENAMED component: update the component name in all tables
- For each LAYER_VIOLATION: update the Layer column to match actual layer
- For each NEW_FLOW: append a row to the Data Flow table
- For each REMOVED_FLOW: remove its row from the Data Flow table
- Update the generation date in the header: `_Generated by /ce:diagram on {today's date}...`

**Update `architecture.excalidraw`:**
Read the current `architecture.excalidraw` (or start with an empty elements array if it does not exist). Use the Excalidraw schema in `references/excalidraw-format.md` to apply changes:

- For each ADDED component: create a new rectangle element + text label. Place it in the correct layer Y position within the feature area frame. If no frame exists for that feature area, create one.
- For each REMOVED component: set `isDeleted: true` on the component's rectangle, label, and any bound arrows.
- For each RENAMED component: find the component's text label element and update its `text` field.
- For each NEW_FLOW: create a new arrow element with correct `startBinding` and `endBinding`. Add `boundElements` backlinks to both source and target shapes.
- For each REMOVED_FLOW: set `isDeleted: true` on the arrow element. Remove `boundElements` entries from the source and target shapes.

Validate all bindings after applying changes (per `references/excalidraw-format.md` binding consistency rules) before writing the file.

**Append to `architecture-changelog.md`:**

Prepend a new entry at the top of the file (create the file if it does not exist):

```markdown
## {YYYY-MM-DD} — /ce:diagram:update
- Added: {component names}
- Removed: {component names}
- Renamed: {from -> to}
- Layer corrections: {component (old -> new)}
- New flows: {from -> to}
- Removed flows: {from -> to}
```

Only include categories that have entries.

### Phase 7: Print Completion

```
Diagram updated. {N} changes applied.

Files updated:
  architecture-constraints.md
  architecture.excalidraw
  architecture-changelog.md
```
